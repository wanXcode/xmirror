# 微信公众号归档回退说明

日期：2026-03-11

## 结论

已将“微信公众号文章归档”相关改动从本地、GitHub、x1 线上全部回退。

当前 XMirror 恢复为：
- 仅支持 X / Twitter 链接归档
- 不支持 `mp.weixin.qq.com` 文章归档

## 回退原因

核心问题不在页面解析逻辑，而在**线上抓取出口被微信风控**：

- 本地环境对部分公众号文章可以抓到正文
- x1 服务器环境多次实测返回的是：
  - 未知错误
  - 环境异常
  - 去验证
  - 暂无权限查看此页面内容
- 即使补齐依赖并复用 `web-content-fetcher` / `scrapling` 思路，x1 仍然拿不到真实正文 DOM

这意味着：

1. 线上问题不是普通代码 bug
2. 在当前服务器出口条件下，公众号归档无法稳定成立
3. 继续把“异常页/权限页”当作正文入库会污染历史记录与缓存

因此决定整体回退，而不是保留半可用实现。

## 已验证事实

- 线上回退后，提交公众号链接会返回：`无效的X链接`
- 首页相关文案也已恢复为仅支持 X / Twitter
- GitHub `main` 与 x1 线上已同步到回退后的版本

## 后续如果要重新尝试

只有在以下前提下才建议重启该需求：

- 引入浏览器真实会话抓取方案
- 或接入外部稳定正文提取服务

否则不建议再次直接在当前 x1 上尝试公众号直抓。
